# 上架准备：主体变更与 QQ 端说明

> 整理日期：2026-07-19 ｜ 技术栈：uni-app + 微信云开发
> 用途：小程序上架前的主体、备案、多端策略决策依据

## 摘要（三句话版）

1. **QQ 端不用做了。** 腾讯已于 2026 年 7 月 1 日公告 QQ 全面接入微信小程序，只需做微信小程序一个，QQ 用户可直接在 QQ 内搜索打开。
2. **主体可从个人变更为企业，但不可逆。** 官方支持「个人迁至组织」，不支持反向迁回，总花费约 1100–1600 元。
3. **顺序很重要：先变更主体，再做备案。** 做反会导致备案作废、要注销重来，白等两三周。

---

## 一、小程序主体变更（个人 → 企业）

微信官方**支持个人主体迁至组织主体**，但不支持个人迁个人、组织迁个人。这是一扇**单向门**：迁到公司名下后就再也迁不回个人名下，日后若要变更只能迁到另一个企业主体。

> **决策提示**：若使用家人或亲友公司的营业执照，需先想清楚长期归属——公司若注销、或将来希望独立运营，小程序归属会很被动。这是权属问题不是技术问题，建议提前与执照方达成书面共识。

**入口**：小程序后台 → 设置 → 基本信息 → 主体信息 → 小程序主体变更。
**流程**：同意须知 → 信息填写 → 授权验证 → 支付费用 → 平台审核。
需上传原主体证件、目标主体证件、迁移申请函，并选择是否同时变更管理员与昵称。**主体变更不影响用户数据**，小程序仍保留在用户的「我的小程序」列表中。

| 费用项目 | 金额 | 说明 |
|---|---|---|
| 主体变更审核服务费 | 300 元 / 次 | 无论审核是否通过**均不退款** |
| 微信认证费 | 300 元 / 年 | 原个人主体若未做过微信认证，变更后需另付 |
| 申请函公证费 | 约 500–1000 元 | 各地公证处收费不同，线下一般 3–5 个工作日 |
| **合计** | **约 1100–1600 元** | 首次办理的现实预算 |

### 关键材料：申请函 + 公证书

《小程序主体变更申请函》须由公证处对印鉴做公证。公证前必须逐字核对：**小程序原始 ID**（不是 AppID）、原主体与目标主体的名称及证件号、双方管理员信息、日期——**错一个字就要重新公证，公证费不退**。另需双方主体证件彩色原件照片、法人身份证正反面、公章。

### 提高通过率：防驳回清单

驳回主要集中在**材料形式问题**而非资质判断，逐条核对可显著提高一次通过率：

- [ ] 申请函用后台**当前最新模板**，旧版直接驳回
- [ ] 多页文件加盖**骑缝章**
- [ ] 证件照须**彩色原件**，黑白复印、模糊翻拍会被拒
- [ ] 公章样式与主体备案信息**完全一致**
- [ ] 公证前逐字核对申请函全部信息
- [ ] 绑过微信支付的，审核通过后 **15 天内解绑**，超时作废
- [ ] 审核期 30 天内有 **3 次**补充机会，别一次用完
- [ ] 变更后原服务类目失效，需用新主体资质重新提交

> **时机优势**：「小程序存在未处理投诉」是驳回原因之一。本项目尚未上线、无用户投诉，是主体变更最干净、通过率最高的时间窗口。

## 二、QQ 端策略：不需要单独开发

> **结论**：**不需要注册 QQ 小程序账号，不需要单独备案，不需要维护第二套代码。** 只做微信小程序一个，QQ 用户即可在 QQ 客户端内直接搜索并打开。

| 时间 | 腾讯平台战略调整 |
|---|---|
| 2025-05 | QQ 9.1.70 版本开启小范围测试，QQ 支持运行微信小程序 |
| 2025-07 | 媒体报道微信与 QQ 小程序打通，超 90% 主流微信小程序可在 QQ 顺畅使用 |
| **2026-07-01** | **腾讯正式公告：QQ 全面接入微信小程序，建议开发者尽快迁移** |

公告要点：目的是「降低开发者双端开发量和小程序维护成本」，**一次开发即可覆盖微信、QQ 两端**；存量 QQ 小程序目前不受影响、仍可更新，但平台已提供 QQ openid 获取接口与用户通知能力，专门用于帮开发者迁移用户资产；迁移非强制，由开发者自行决定。综合判断：**QQ 小程序平台正在有序收尾，新项目不应再投入 QQ 端开发。**

**用户怎么找到**：QQ 顶部搜索框点击「小程序」，搜名称即可，图标右下角带绿色角标的表示来自微信。首次打开会跳转微信授权一次，此后打开其他微信小程序均无需再授权。目标用户是 QQ 群跑团玩家，首次授权需装有微信，对该群体基本无门槛。

> **注意**：QQ 开放平台文档站（q.qq.com/wiki）中仍保留「QQ 小程序与微信小程序账号不互通、需分别注册」等**存量表述**，文档未随战略调整更新。查阅时以官方公告为准，而非文档站内容。

## 三、执行顺序

1. **确认企业营业执照可用、法人配合** — 这一步卡住的话后面都免谈
2. **小程序主体变更（个人 → 企业）** — 期间小程序照常开发，不受影响
3. **微信认证 + 重新提交服务类目** — 变更后原类目失效，需用新主体资质重报
4. **提交小程序备案** — 必须在主体变更完成之后
5. **提审上架** — 备案号下发后方可发布版本

### 为什么不能先备案

当备案的**主办单位名称和统一社会信用代码都变更**时，微信不支持走「变更备案」流程，必须**先完成主体变更 → 注销当前备案 → 用新主体重新发起备案**。也就是说，若在主体变更前先做了备案，这份备案会整个作废重来，白白浪费两到三周审核周期。

- 仅名称变更、统一社会信用代码未变的情况，可以直接走变更备案流程
- 因主体变更而注销备案的，注销期间暂不影响小程序使用，但应尽快用新主体重新发起；**注销备案不可逆**

| 备案环节 | 耗时 |
|---|---|
| 备案信息填写 | 当天 |
| 平台初审 | 1–2 个工作日 |
| 工信部短信核验（12381 发送，须 24 小时内完成） | 当天 |
| 各省通信管理局审核 → 下发备案号 | 1–20 个工作日 |

## 四、其他上架前置事项

### 服务类目

**切勿选择「游戏」类目**——要求提供游戏版号，普通开发者无法取得，必选必拒。本产品是「组局信息发布工具」不是游戏本体，此区分需在类目和服务描述中明确体现。建议优先报**「工具」类目**，与产品实际定位（发布编辑器为主、浏览大厅为辅）相符。

### UGC 合规能力（当前代码尚未实现，属开发工作量）

本产品含用户生成内容（招募标题与描述、昵称与签名、头像、演绎记录），微信对此类小程序有硬性要求，缺失会导致驳回：

1. **内容安全检测**——用户提交的文字、图片需调用微信内容安全接口校验
2. **举报入口**——招募详情页、用户主页需提供举报违规内容的入口
3. **拉黑 / 屏蔽功能**——用户需能屏蔽特定用户，屏蔽后不再看到其内容

### 隐私指引与域名

产品收集头像、昵称、QQ 号 / 微信号，需在后台填写《用户隐私保护指引》声明，否则相关接口调用会被限制。另外，本项目采用微信云开发架构，**小程序备案不需要域名和服务器**；已有的域名 / 服务器可留作官网用途，那种情况才涉及传统网站 ICP 备案。
