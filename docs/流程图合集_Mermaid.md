## 五、发布人用户旅程图

```mermaid
flowchart TD
    S["打开小程序\nP1 招募大厅"] -->|"点击悬浮 +"| T["触发登录检查"]
    T -->|"已登录"| F["填写发布表单\nP3 · 目标 < 10 秒"]

    F --> D{"保存还是发布？"}
    D -->|"保存草稿"| DR["草稿已保存\nP8 我的创建\n状态：draft"]
    D -->|"发布（弹'发出去喽'确认）"| LV["招募上线！\nP8 我的创建\n状态：recruiting"]

    LV -->|"等待申请（焦虑期）"| W["收到申请\nP8 / P9 查看"]
    W -->|"进入审核"| RV["审批申请人\nP10 同意/拒绝"]
    RV -->|"审批完毕"| DP["发车！\nP9 确认后 → P8\n状态：finished"]
```

### 发布人情绪曲线

| 阶段 | 情绪 | 说明 |
|---|---|---|
| 打开 → 填表单 | 顺畅 | 极简流程，10 秒完成 |
| 保存 or 发布 | 犹豫 | 信息填完了吗？要不要先存草稿？ |
| 等待申请 | 焦虑 | 有没有人来？需不需要去分享？ |
| 审批申请人 | 投入 | 这个人靠谱吗？看看档案馆 |
| 发车 | 满足 | 人齐了，出发！ |

## 六、申请人用户旅程图

```mermaid
flowchart TD
    B["浏览招募大厅\nP1 · 无需登录"] -->|"筛选/搜索 → 点击卡片"| V["查看招募详情\nP2 · 无需登录"]
    V -->|"点击发布人头像（需登录）"| C["查看发布人主页\nP4 · 档案馆信任评估"]
    C -->|"返回 P2"| V

    V -->|"点击'申请加入'（需登录）"| LK{"乐观锁检查\n招募仍为 recruiting？"}
    LK -->|"是"| P["申请已提交\n状态：pending"]
    LK -->|"否"| X["弹提示：已发车\n按钮置灰"]

    P -->|"等待审核（焦虑期）\nP8 我的申请 · 主动轮询"| R{"审核结果？"}
    R -->|"approved 已通过"| G["获取联系方式！\nP2 点击绿色'已通过'按钮\n弹窗显示联系方式"]
    R -->|"rejected 未通过"| NG["未通过\nP2 按钮置灰"]
```

### 申请人情绪曲线

| 阶段 | 情绪 | 说明 |
|---|---|---|
| 浏览大厅 | 随意 | 随便看看有没有合适的车 |
| 查看详情 + 档案馆 | 好奇 | 这个 KP 靠谱吗？看看 ta 的跑团记录 |
| 点击申请 | 期待 | 希望能上车 |
| 等待审核 | 焦虑 | 会通过吗？什么时候出结果？ |
| 收到结果 | 释然或失望 | 通过了就拿联系方式，没通过就继续找 |
