## 1. 基本信息

| 属性 | 内容 |
|---|---|
| 产品定位 | QQ 群跑团生态的高效率组局插件（非跑团工具本体） |
| 适用端 | 微信小程序 / QQ 小程序（双端互通） |
| 北极星指标 | 发布人完成一次招募发布的时间 < 10 秒 |
| 核心理念 | 极简，不改变玩家原有 QQ 群跑团习惯 |
| 技术栈 | uni-app + Vue3 + Vite + Pinia（JavaScript） |

## 2. 用户角色

### 2.1 发布人（GM 或 PL 均可）

核心诉求：快速发车，招募到靠谱的玩家。

**最高频动作：**

- 发布招募（极简表单，目标 10 秒内完成）
- 审批申请人：查看申请人列表 → 点击进入申请人主页 → 同意或拒绝
- 同意申请后：系统自动将联系方式展示给申请人（申请人主动查看）
- 手动点击「发车」按钮，将招募状态改为已发车

### 2.2 申请人（PL 或 GM）

核心诉求：快速找到符合时间和规则体系的车，顺利上车。

**最高频动作：**

- 浏览与筛选（在招募大厅中按 tag 找车）
- 查看发布人主页及档案馆，判断是否值得申请
- 报名上车（申请加入）
- 在「我的申请」中主动轮询查看审批结果
- 审批通过后：点击详情页底部按钮弹出发布人联系方式

## 3. 登录策略

所有需要身份的操作统一跳转至 `pages/auth/login`，微信一键授权，授权完成后返回原页面。

| 场景 | 是否需要登录 |
|---|---|
| 浏览招募大厅 | 否，匿名可访问 |
| 查看招募详情页 | 否，匿名可访问 |
| 点击发布人头像进入用户主页 | 是，触发微信授权 |
| 点击「申请加入」按钮 | 是，触发微信授权 |
| 进入 Tab2 快速发车 | 是，进入即触发 |
| 进入 Tab3 我的 | 是，进入即触发 |

## 6. 招募卡片状态机

| 状态 | 触发条件 | 可执行操作 |
|---|---|---|
| draft（草稿） | 发布人保存表单但未发布 | 编辑、发布（→ recruiting） |
| recruiting（招募中） | 发布人点「保存及预览」确认发布 | 编辑、审核、发车（→ finished） |
| finished（已发车） | 发布人手动点「发车」并确认 | 仅只读，不可逆 |

> 架构备注：状态字段为 `status`（string），预留未来在「我的创建」列表顶部增加 Tab 筛选（全部 / 招募中 / 草稿 / 已发车）。

## 7. 页面结构详细定义

### 7.1 招募大厅 `pages/home/index.vue`

**重要规则：只展示 status = recruiting 的卡片，已发车卡片不在此页出现。**

#### 筛选栏（吸顶，横向滚动）

| 筛选维度 | 选项 |
|---|---|
| 规则体系 | COC / DND / 其他（非 COC/DND 均归入「其他」） |
| 招募对象 | KP / PL / DM |
| 跑团方式 | 语音 / 文字 |

#### 搜索框

按模组名称搜索，按回车或点搜索按钮触发，不做实时搜索。

#### 招募卡片内容

| 元素 | 说明 |
|---|---|
| 封面图 | 100×100rpx，圆角 10rpx，无图则显示占位色块 #d9d9d9 |
| 模组名称 | 粗体，36rpx，主标题 |
| Tag 行 | 跑团方式 / 规则体系 / 招募人数 |
| 信息行一 | 预计时长 │ 招募对象 |
| 信息行二 | 跑团时间 │ 招募人 ID（右对齐，#b2b2b2） |
| 右下角人数 | 招募中状态显示「X 人申请」；已发车状态不显示人数 |

---

### 7.2 招募详情页 `pages/home/detail.vue`

此页面统一使用申请人视角。发布人从大厅点击自己的卡片也看到此视角；从「我的创建」列表进入则跳转发布人详情页（7.7 节）。

#### 页面结构（从上到下）

- 顶部导航栏：模组名称（居中）+ 返回按钮
- 封面图（100×100rpx）+ 发布人头像/昵称 → 点击触发登录后跳转用户主页
- Tag 行：跑团方式 / 规则体系 / 人数
- 信息行：预计时长 / 招募对象 / 跑团时间
- 分割线 + 「模组简介」标题
- 简介正文（长文，支持长按复制）
- 底部吸底按钮（五态）

#### 底部按钮五态

| 访问者身份 | 申请状态 | 按钮文字 | 样式 | 点击行为 |
|---|---|---|---|---|
| 发布人本人 | — | 管理招募 | 深色宽按钮 | 跳转 pages/mine/detail |
| 申请人/游客 | 未申请 | 申请加入 | 深色宽按钮 | 未登录→授权页；已登录→提交申请 |
| 申请人 | 待审核 | 等候审核中 | 灰色，不可点 | 无响应 |
| 申请人 | 已通过 | 已通过 | 绿色宽按钮 | 弹出联系方式弹窗 |
| 申请人 | 已拒绝 | 未通过 | 灰色，不可点 | 无响应 |

> 发布人本人点击「申请加入」：弹出提示「你是本次招募的发布人，无法申请加入」，不做状态变更。

#### 联系方式弹窗

- 标题：你已通过审核
- 内容：发布人预设的联系方式（微信号 / QQ 号 / QQ 群号 + 具体号码）
- 按钮：我知道了（关闭弹窗）

> 联系方式**只在此弹窗里展示**，不在任何列表卡片上直接显示，保护发布人隐私。

### 7.3 快速发车 / 编辑表单 `pages/publish/index.vue`

进入 Tab2 立即触发登录。有 `id` 参数 = 编辑模式（回填数据）；无 `id` 参数 = 新建模式。提交后跳转至 `pages/mine/created?tab=created`。

#### 表单字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 模组封面 | 图片上传 | 否 | 占位色块，未来支持上传 |
| 模组名称 | 文字输入 | 是 | 不超过 20 字；旁边预留「导入模组信息」按钮（MVP 点击显示「即将开放」，V1.1 接入 D 站/魔都自动填充名称+简介） |
| 预计时长 | 时间选择器 | 是 | 1 小时 ~ 99 小时 |
| 跑团时间 | 时间选择器 | 是 | 开始-结束时间 |
| 跑团规则 | 单选 | 是 | COC6th / COC7th / DND5E / FATE / Pathfinder / 其他（注：大厅筛选归并为 COC/DND/其他三类） |
| 跑团方式 | 单选 | 是 | 语音 / 文字 |
| 跑团人数 | 数字输入 | 是 | 总人数 |
| 招募 PL | 数字输入 | 否 | 缩进子项 |
| 招募 KP/DM | Toggle 开关 | 否 | 缩进子项 |
| 模组简介 | 多行文本 | 否 | 不超过 300 字 |
| 邀请方式 | 单选 + 文本输入 | 是 | 微信号 / QQ 号 / QQ 群号，三选一；选 QQ 号时自动带入个人资料中预存的 QQ 号，可临时覆盖修改；提示文案：「审核通过后将自动发送给申请人」 |

底部吸底按钮：**保存及预览**（深色宽按钮）

### 7.5 演绎记录详情页 `pages/profile/log_detail.vue`

同一页面承担查看和编辑两种模式。他人访问为纯只读，无操作按钮。

#### 编辑态字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 标题 | 单行文本 | 是 | |
| 模组名称 | 单行文本，自由输入 | 是 | 浏览态降格为 tag 与其他 tag 并排展示 |
| tag | 预设列表 + 自定义输入，多选 | 否 | 预设如：COC / DND / 语音团 / 文字团等 |
| 正文 | 多行长文本 | 是 | 支持直接粘贴长文字 log |

#### 浏览态展示结构

```
标题（大字粗体）
模组名称 · tag1 · tag2  ← 统一以小字灰色显示在标题下方
正文全文
```

#### 双态切换规则

| 访问者 | 交互规则 |
|---|---|
| 他人访问 | 纯浏览态，无任何操作按钮 |
| 本人 - 浏览态 | 右上角「编辑」按钮，点击切换为编辑态 |
| 本人 - 编辑态 | 右上角「保存」按钮，保存后返回浏览态 |

### 7.6 我的创建 / 申请 / 足迹 `pages/mine/created.vue`

⚠️ **以下内容以补充说明为准（优先级高于主文档）**

同一个页面承担三个入口，通过 URL 参数 `tab` 区分。三种状态卡片（草稿/招募中/已发车）混排，按时间倒序，MVP 不做筛选，但 `status` 字段预留。

#### 三个 tab 的本质

| Tab | 本质 | 数据来源 | MVP 状态 |
|---|---|---|---|
| 我的创建（created） | 我发布的招募 | 自己创建的全部招募（含草稿/招募中/已发车） | 真实可用 |
| 我的申请（applied） | 我报名的招募 | 我提交过申请的全部招募，不管审核结果 | 真实可用 |
| 我的足迹（history） | 我浏览过的招募 | 浏览历史记录 | 占位，点击 Toast「即将开放」，MVP 不做 |

> 用户可以同时在「我的创建」和「我的申请」中都有卡片，两个 tab 完全独立，互不干扰。

#### Tab 视图跳转逻辑

| tab 参数 | 展示内容 | 卡片点击跳转 |
|---|---|---|
| created | 我发布的所有招募（全状态） | pages/mine/detail?id=xxx（发布人详情页） |
| applied | 我申请的招募 | pages/home/detail?id=xxx（申请人视角详情页） |
| history | 我的足迹（浏览历史） | pages/home/detail?id=xxx（申请人视角详情页） |

#### 「我的创建」卡片右下角

| 招募状态 | 右下角显示 |
|---|---|
| draft（草稿） | 不显示 |
| recruiting（招募中） | X 人申请（#b2b2b2 灰色） |
| finished（已发车） | 不显示 |

#### 「我的申请」卡片右下角

显示**我的申请状态**（不是招募本身的状态）：

| 申请状态 | 标签文字 | 文字颜色 | 背景色 |
|---|---|---|---|
| pending（审核中） | 审核中 | #888888 | #f0f0f0 |
| approved（已通过） | ✓ 已通过 | #2a7a4a | #e8f5ee |
| rejected（未通过） | 未通过 | #666666 | #e8e8e8 |

**已通过卡片的联系方式获取路径：**  
点击卡片 → 进入申请人视角详情页 → 底部按钮显示「已通过」（绿色）→ 点击弹出联系方式弹窗

### 7.7 发布人详情页 `pages/mine/detail.vue`

从「我的创建」tab=created 点击卡片进入。页面内容与申请人详情页复用 ModuleDetail 组件，底部按钮不同。

#### 底部按钮随状态变化

| 卡片状态 | 底部按钮 | 操作说明 |
|---|---|---|
| draft（草稿） | 编辑 / 发布 | 发布→状态变 recruiting，卡片上线 |
| recruiting（招募中） | 编辑 / 审核 / 发车 | 审核→跳 pages/mine/review；发车→弹确认窗 |
| finished（已发车） | 已发车（灰色，不可点） | 页面只读，不可操作 |

#### 发车确认弹窗

内容：「当前已通过 X 人，发车后招募将关闭，确认发车？」  
按钮：确认发车 / 取消

### 7.8 审核页 `pages/mine/review.vue`

从发布人详情页点击「审核」按钮跳转进入。仅发布人本人可访问。

#### 申请人列表条目结构

- 头像（点击跳转该申请人用户主页，只读态）
- 昵称
- 申请时间
- 同意 / 拒绝 按钮（已处理后变为状态标签）

#### 操作后状态变化

| 操作 | 列表变化 |
|---|---|
| 点击「同意」 | 按钮区替换为「已通过」绿色标签，条目保留 |
| 点击「拒绝」 | 按钮区替换为「已拒绝」灰色标签，条目保留 |

> MVP 不做推送通知，申请人主动轮询状态，无需订阅消息权限。

## 10. 技术规范

### 10.1 页面跳转规范

| 场景 | 正确写法 | 错误写法 |
|---|---|---|
| 跳转 TabBar 页面 | `uni.switchTab({ url: ... })` | `uni.navigateTo`（报错） |
| 跳转普通页面 | `uni.navigateTo({ url: ... })` | — |
| 返回上一页 | `uni.navigateBack()` | — |

### 10.2 微信小程序 CSS 规范

- 组件样式只允许 class 选择器，禁止标签选择器（`view{}`）、ID 选择器、属性选择器
- 组件必须单根节点，不支持 fragment（多根节点）
- 禁止使用 CSS 通配符 `*`
- 禁止使用 `vh` / `vw` 单位，改用 `%` 或 `rpx`
- 所有尺寸单位统一使用 `rpx`

### 10.3 图片资源规范

- 禁止引用 `/static/` 目录下不存在的本地图片
- 所有图片占位一律使用背景色块（`#d9d9d9`）代替
- 真实图片路径通过接口数据动态传入，不写死在代码里
